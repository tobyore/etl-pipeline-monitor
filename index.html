🔄 What I Iterated On

Iteration 1: Noisy ADF Alerts

Problem: Azure Data Factory fired duplicate alerts for cascading pipeline failures — a single source table timeout triggered 12 downstream pipeline alerts, flooding the Teams channel.

Fix: Added a deduplication layer using Azure Table Storage. The workflow hashes the pipeline name + error code and checks for matching entries within a 20-min window. Reduced alert noise by 73%.

Iteration 2: AI Misclassifying Schema Drift

Problem: Azure OpenAI classified "column mismatch in member_profiles sink" as an Infrastructure issue instead of Schema Drift. This routed tickets to the wrong team.

Fix: Added few-shot examples specific to ABC Fitness data models (member profiles, billing, check-ins) and a confidence threshold — if AI confidence <80%, it flags for human review on the Data Governance team instead of auto-routing.

Iteration 3: Slack Channel Noise

Problem: Posting every pipeline status update to a single #etl-incidents channel made it impossible for engineers to track what mattered. Critical alerts got buried under resolved/info messages.

Fix: Split into severity-based channels (#etl-incidents-critical, #etl-incidents-high) and added threaded updates — initial alert posts to channel, all follow-up status changes reply in-thread. Reduced engineer context-switching by 40%.

🏗️ Technical Architecture

Trigger

Azure Data Factory diagnostic logs → Azure Monitor Alert Rule → Logic App webhook

POST /api/etl-monitor/webhook

AI Classification

Azure OpenAI (GPT-4) via Azure Function

Prompt: "Classify this ADF failure: {error}. Return: severity, root_cause, owning_team"

Deduplication

Azure Table Storage (pipeline fingerprint lookup)

Filter: skip if matching hash in last 20 min

Routing & Notification

Microsoft Teams + Jira Cloud (async via Service Bus)

#etl-incidents-{severity} channel + auto-assigned Jira ticket
